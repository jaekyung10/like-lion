# Week1 HTML&CSS 기초 과제

## 모르는 부분 정리

### 1. layout1
- 이미지를 가져와 CSS를 적용할 때 `background-image`, `background-size`만 설정해도 되는지 궁금했다.
- 현재는 이 두 속성만으로도 화면에 이미지는 잘 보이지만, 상황에 따라 추가로 설정해야 하는 속성이 있는지 알고 싶다.

### 2. layout2
- 클래스 이름을 `footer_left`, `footer_right`처럼 작성해도 괜찮은지 궁금했다.
- `header`에서 사용하는 `logo`, `nav`처럼 자주 쓰이는 이름이 따로 있는지,
  혹은 내가 임의로 왼쪽, 오른쪽, 위쪽, 아래쪽 기준으로 이름을 정해도 되는지 헷갈렸다.

### 3. layout3
- layout3는 이전 레이아웃보다 나누어야 할 영역이 많아 구조를 설계하는 과정이 특히 헷갈렸다.
- `box1`, `box2`, `box3`처럼 이름을 붙여 구분했는데, 이런 방식으로 클래스 이름을 작성해도 괜찮은지 궁금했다.
- 또한 요소들 사이에 간격이 있는 경우, 그 차이를 `width`, `height`로 조절해야 하는지,
  아니면 `top`, `left` 같은 위치값으로 조절해야 하는지 정확히 구분이 되지 않았다.
- 이 과정에서 `relative`, `absolute`를 사용해야 하는지도 잘 모르겠고,
  사용해야 한다면 어떤 기준으로 적용해야 하는지 아직 명확히 이해하지 못했다.
- 그래서 우선은 임의로 간격을 설정해 최대한 비슷하게 구현했지만,
  이 방법이 정확한 방식인지는 확신이 들지 않아 더 올바른 기준을 배우고 싶다.

### 4. layout4
- layout4에서도 클래스 이름을 `bot1`, `bot2`처럼 나누어 사용했는데,
  이런 식으로 이름을 구분하는 방식이 괜찮은지 궁금했다.

## 느낀 점
- 이번 과제를 하면서 HTML 구조를 처음 설계할 때 전체 구조를 먼저 잘 나누는 것이 매우 중요하다는 점을 느꼈다.
- 특히 클래스 이름도 단순히 만드는 것이 아니라, 나중에 다시 보았을 때 구조가 한눈에 이해될 수 있도록 체계적으로 짓는 방법이 중요하다고 생각했다.
- 처음에는 구조를 나누고 배치하는 과정이 낯설어서 시간이 오래 걸렸지만,
  직접 하나씩 구현해 보면서 점점 더 익숙해졌고, 갈수록 재미를 느낄 수 있었다.
